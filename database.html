PROJECT · 2015–2016

AGV 小车 — 车道线识别与自动避障

01项目概述

基于 Arduino Mega 2560 与 Raspberry Pi 双控制器架构的 AGV（自动导引）小车，配备双摄像头分别承担循迹与避障任务，使用麦克纳姆轮实现全向移动，通过增量式 PID 控制电机 PWM 完成车道线跟随与障碍物绕行。

主控Arduino Mega 2560 + Raspberry Pi（图像处理）
驱动BTS7960 H 桥驱动板 × 4
底盘四麦克纳姆轮全向底盘
感知循迹摄像头 + 避障摄像头 + 超声波
通信Pi ↔ Arduino 串口（USB / 9600 bps）

02开发历程

从早期单层亚克力面包板原型机起步，逐步替换为麦克纳姆轮底盘并加装树莓派与双摄像头，最终形成可在户外石板路稳定循迹避障的整车。

03整车结构（图 6.4 小车实物图）

04总体设计方案（图 1.1）

图像采集 → 树莓派预处理 → 障碍物判别 → 增量式 PID → 串口通信 → Arduino 接收 → 改变电机 PWM 波 → 车身方向控制。检测到障碍物时切换至 Arduino 端避障逻辑。

05硬件接线 · 树莓派 / Arduino / 驱动板

树莓派共有 4 个 USB 端口、1 个 Micro USB 端口。其中三个 USB 端口任意接 2 个网络摄像头及与 Arduino Mega 2560 相连的串口线；Micro USB 端口接入外接电源（5V 2A）。

Arduino Mega 2560 的 PWM 输出口（2、3、4、5、6、7、10、12）分别接入驱动板电机驱动端口（A1、A2、B1、B2、C1、C2、D1、D2），每一对端口对应每个电机的前转 / 后转。驱动板 VCC、EN、GND 分别由 Arduino 的 5V、3.3V、GND 提供。

06树莓派 ↔ Arduino 软件通信

两端均通过 USB 实现串口通信，数据以字符串形式传输。树莓派端使用 Python serial 库，调用 ser.write() 发送；Arduino 端通过 Serial.available() 判别有效数据，并以 Serial.read() 逐字节读取。

Python · 树莓派端

# 检查转化成字符的变量值能否通过串口通信传给 arduino

ser = serial.Serial('/dev/ttyUSB0', 9600)

t = 10
output = t * 10
output = "fr" + str(output)
x = output.encode('utf-8')

无线充电小车 — 超级电容供能 · 断电自启动

01项目概述

2018 年上海市大学生电子设计竞赛（TI 杯）参赛作品。整车以超级电容代替电池，通过无线方式从外接 5V/1A 直流稳压电源取电；当无线供电断开时，继电器触发自启动逻辑，DC-DC 升降压电路维持 3.3V 输出，驱动小车继续行驶直至电容耗尽。

PROJECT · 2018 · 校内赛

电磁导航智能车 — 双电感差分循迹

01项目概述

全国大学生智能汽车竞赛 · 电磁组（校内赛）。赛道中央铺设 20 kHz / 100 mA 通电导线，车头横杆两端的工字电感采集交变磁场，经放大检波后通过差分算法解算车体相对赛道中心线的偏差，由 PID 控制器闭环调节舵机转向与电机 PWM，实现高速循迹。

主控NXP MCU（智能车竞赛标配）
检测原理双电感差分 · 工字电感 + 运放放大 + 检波
赛道信号20 kHz · 100 mA 交变电流（赛规标准）
控制算法位置式 / 增量式 PID（转向 + 速度）
赛道元素十字路口判断 · 起跑线检测
底盘 / 供电B 车模 · 7.2V Ni-Cd 电池组

02感应板与差分原理

两枚 10 mH 工字电感对称布置在车头横杆两端，采集到的交变电压经选频放大、二极管检波后转为直流电平送入 MCU 的 ADC。当车体偏离中心线时，靠近导线一侧的电感感应电压上升、远离一侧下降；用 (L−R)/(L+R) 做归一化差分即可消除整体磁场强度波动的影响，得到与偏差量近似线性的控制信号。

差分归一化 · Error Calculation

// 双电感差分 · 消除幅度漂移
float L = adc_read(LEFT);
float R = adc_read(RIGHT);
float err = (L - R) / (L + R);
// err ∈ [-1, +1]，0 即车身居中

位置式 PID · Steering Loop

float pid_steer(float err) {
    integral += err;
    float d = err - last_err;
    last_err = err;
    return Kp*err + Ki*integral + Kd*d;
}

自定时炸弹 — 8086 + 8255/8253/8259/8279

01项目概述

用 8086 汇编实现一枚带按键解除的"自定时炸弹"。开机后用 8279 键盘选定炸药量（决定倒计时长度与最终爆炸效果），数码管同步显示剩余秒数，倒数过程中可随时通过 8259 中断输入解除密码；超时未解除则点亮 8255 控制的 LED 阵列模拟爆炸。整套程序全部在数据手册级别用 IN/OUT 直接编程驱动，没有库函数。

主控8086 · 实模式汇编（MASM）
I/O 扩展8255 并行口（流水灯 + PC0 秒脉冲）
定时8253 双计数器级联 · 模式 2 + 模式 0 → 1 Hz
中断8259 单片 · IR0 上升沿触发解除中断
键盘 / 显示8279 编程键盘控制器 · 8 位共阳数码管 + 16 键
倒计时档位5 s / 10 s / 15 s 三档（按 0/1/2 键选择）

02功能与流程

原始设计来自一张随手画的功能图——倒计时、炸药量自选、最低 30 秒不可解除、最后 10 秒促音、爆炸时 LED 显示爆炸程度、按键输入解除密码。代码版本简化为三档倒计时 + 中断解除 + 三种爆炸花样。

03主控状态机

主程序启动时初始化 8279，循环扫描键盘等待档位输入；输入合法后跳入对应的 CD0/CD1/CD2 倒计时过程。每秒由 8253 + 8255 PC0 协作产生秒脉冲，TEST_C3 检测到上升沿后递减 BCD 计数器并刷新数码管；倒计时期间 SCAN_PW 打开 8259 中断，IR0 触发的 INT_0 读取 8255 PB 端口比对解除密码（这里取 80H 为正确密码），匹配则跳回 START 复位，否则进入 BOOM 直接引爆。

8279 键盘扫描 · Polling Loop

; 主循环：等待两位档位键
START1: CALL SCAN_KEY
        JNC  START1
        XCHG AL,KEYCOUNT
        INC  AL
        CMP  AL,2
        JNZ  START2
        JMP  START1
START2: XCHG AL,KEYCOUNT

; T0: 模式 2，1 ms 方波

OUT DX,AL
; T1: 模式 0，1000 计数 → 1 s
MOV AL,71H
OUT DX,AL
; PC0 由 OUT1 拉高 → 主循环轮询

04中断解除与爆炸效果

解除中断由 WriIntver 把 INT_0 入口写到向量表 8H（IRQ0 在实模式下对应中断号 8）；中断响应里 没有调用 IRET——而是直接 JMP START 复位整机，或 JMP BOOM 强制引爆并清掉调用栈里残留的 5 层 PUSH，避免回到倒计时循环。倒计时正常归零则走 ECD → LIGHT 路径，按选定档位查 LED_Data 表点亮 8255 PA 的 LED 序列（11111B / 00111B / 00000B 模拟由小到大三档爆炸），延时 3 秒后熄灭、HLT。

正确密码从 8255 PB 读到 80H（10000000B）
错误密码任意非 80H · 立即跳 BOOM
归零路径Counter == 0 · 走 LIGHT 流程播放预设花样
三档花样PA = 1FH / 07H / 00H · LED 数量逐档增加

WriIntver     PROC       NEAR
              PUSH       ES

06踩过的坑

① 中断里清栈。因为中断可能从倒计时子程序的深层 CALL 处发生，直接 JMP BOOM 会让 SP 错位；最终在 BOOM 入口连 POP 五次把现场全部丢弃，再读 8255 PA 输出爆炸图样。② 8253 模式选择。T0 用模式 2（速率发生器）保证 1 ms 节拍稳定，T1 用模式 0（计数终止）做 1000 次计数后翻转 OUT1，这两档不能互换。③ BCD 减一。用 SUB AL,1 / DAS 让数码管显示 09 → 08 而不是 09 → 0F。
